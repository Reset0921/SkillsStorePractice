核心问题 1

什么是 Skills?

Skills 是 Anthropic 推出的 Claude Agent 能力扩展标准，让 AI 拥有可复用的「专业技能」

Skill = 装着「专业流程」的文件夹

不是模型参数，不是简单的 Prompt，不是黑盒工具，而是结构化的专业知识包

SKILL.md

使用说明

执行流程

工作步骤

脚本代码

可执行逻辑

模板资源

辅助材料

没有 Skill 的 Agent

像一个 IQ 300 但毫无行业经验的新人

  • 每次都要反复解释背景
  • 行为不稳定、不一致
  • 知识无法沉淀和复用

有 Skill 的 Agent

像一个有 10 年临床经验的专业医生

  • 专业知识已被固化
  • 行为可预测、可复制
  • 经验可传承和共享

本质：经验的封装，知识的资产化

核心问题 2

Skills 有什么用?

从个人效率提升到企业级知识管理，Skills 带来全方位价值

自动化重复任务

将常见的工作流程固化为 Skill，一键触发复杂操作

如：一键创建微应用、自动生成测试用例

团队知识共享

专家经验不再是「老师傅的秘诀」，而是可分享的数字资产

如：代码规范检查、部署流程标准化

保证行为一致性

同样的任务，每次都按照标准流程执行，结果可预测

如：安全审计、合规检查

加速新人上手

新员工通过 Skills 快速掌握团队最佳实践

如：开发环境配置、项目初始化

版本控制与演进

Skill 是文件，可以用 Git 管理、回滚、审计

如：追踪流程变更、A/B 测试不同策略

能力无限扩展

Agent 可以「拥有」上百个 Skills，按需加载

如：多语言翻译、多平台发布

真正稀缺的不是「会思考的 AI」，而是被整理、被固化、能反复调用的专业流程

核心问题 4

Skills 在哪些方面带来提升?

相比传统的 Prompt、Tool、MCP，Skills 在多个维度实现质的飞跃

不占用上下文

Skill 采用「按需加载」的方式:默认只向 Agent 暴露必要的元信息,只有在任务过程中判断需要某项能力时,才会主动读取。

Agent 可以同时「拥有」上百个 Skill

根据当前任务自行决策调用哪一个

可维护

和 Prompt 最大的不同在于:Skill 是文件。文件可以版本控制,可以回滚、演进、审计。

可预测的行为

可复制的能力

可传承的知识

真正的知识沉淀

Prompt 是一次性的对话技巧,Skill 是可复用的操作知识。它记录的不是「你怎么说」,而是「事情应该怎么做」。

结构化的专业知识

可组合、可扩展

具体实例

官方展示了些 Claude Skills,涵盖创意应用到技术任务再到企业工作流程，下面是企业工作中的一个例子的Skill

create-microapp-from-test-template

微应用创建技能

skill.md 结构 

# create-microapp-from-test-template

基于 child-test-manage 模板创建新的微应用。

## 使用方式

在对话中说 "从test模板创建微应用" 或
"/create-microapp-from-test-template" 来触发此技能。

## 模板信息

模板使用 `project/child-test-manage`，配置如下：
- 微应用名称: `test`
- 包名称: `child-test-manage`
- 端口号: `6015`
- 中文标题: `测试微应用`

## 参数说明

执行时会询问以下参数：
参数 | 说明 | 示例 |
|------|------|------|
微应用名称 | 用于路由和 qiankun 注册的标识 | `exam`, `art`, `micro` |
包名称 | package.json 中的 name | `child-exam-manage` |
端口号 | 开发服务器端口 | `6001`, `5100` |
中文标题 | 页面显示的中文名称 | `考试管理` |

## 自动处理的内容

技能会自动：
1. 复制模板目录（跳过 node_modules, dist 等）
2. 修改配置文件（package.json, vite.config.ts/js, presets/index.ts/js 等）
3. 注册到主应用
4. 更新根 package.json

## 后续操作

创建完成后：
1. 运行 `yarn install` 或 `npm install` 安装依赖
2. 运行 `yarn <微应用名>` 启动微应用
3. 访问 `http://localhost:<端口>` 或主应用的 `/<微应用名>` 路由

核心功能

复制模板目录

将 child-test-manage 目录复制为新名称（跳过 node_modules, dist 等）

修改配置文件

更新 package.json, vite.config.ts/js, presets/index.ts/js, index.html, src/main.ts/js, src/plugins/router.ts/js, src/style/base.less

注册到主应用

在主应用 config 中添加微应用注册

更新根 package.json

添加开发脚本和 workspace

设计亮点

这个目录体现了 Skill 在企业级开发中的价值：将复杂的微应用创建流程标准化、自动化。通过 skill.md 文件清晰地定义了使用方式、参数说明、自动处理内容和后续操作，让开发者可以快速创建新的微应用，避免手动修改多个配置文件的繁琐过程。

自动化流程 配置管理 模板复用 企业级应用

Skill 未来方向

Testing & Evaluation

可以像现在的软件一样进行测试、分析 Skill。确保技能的质量和可靠性。

Versioning

新增版本控制,比如像现在的 node 包版本一样。支持技能的版本管理和升级。

Skill dependencies

Skill 引入另一个 Skill,互相组合依赖,构建更加强大的 Skill。形成技能生态系统。

安全提示

技能通过指令和代码为 Claude 提供新功能。虽然这使它们变得强大,但也意味着恶意技能可能会在它们被使用的环境中引入漏洞,或指示 Claude 窃取数据并采取非预期行动。

我们建议仅从可信来源安装技能。在从不太可信的来源安装技能时,在使用前应彻底审核。首先,阅读技能捆绑文件的内容,了解其作用,特别关注代码依赖和捆绑资源(如图像或脚本)。同样,注意技能中的指令或代码,这些指令或代码指示 Claude 连接到可能不受信任的外部网络来源。

注意: 社区已经看到有人遇到过 Skills 投毒的情况,请务必谨慎对待第三方 Skills。
